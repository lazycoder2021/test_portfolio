Creating Collections Using MongoDB

Overview

Objective is to create 2 collections (for storing data) using MongoDB shell (command line interface).

What is a Collection in MongoDB?

To understand this, let's take a look at the structure of MongoDB:

As you can see, every MongoDB is divided into many collections which in turn can contain many documents. Documents are the real deal. They contain our data in json format. That is, key:value pairs. Whereas, collections allow us to segregate that data and structure them properly.

For example, let's say 'users' database has to store users from across the world. That means it has to store users from India, China, USA etc. Storing all of them in one place can cause confusion. To avoid this, the user data can be stored separately based on the country they are from using collections.

To create MongoDB documents (the data), we first have to create a collection for storing those documents.

How to Create Collections using MongoDB?

Follow the instructions given below:

  1. Using 
    Windows + R
    keys and typing 'mongo', access the mongo shell.
  2. MongoDB console (Mongo Shell to be more precise) will open up as shown below.
  3. Clear the screen (using 'cls' command) so that you can begin without any distraction.
  4. In the blank screen, create a new database using mongodb command: use users
    • This will create a new database called 'users' and MongoDB gets 'switched to db users' now.
    • You can also check whether 'users' is indeed your current database by typing db
    • db always refers to your current database. You will use this command regularly.
  5. To create a new collection in 'users' db, you should use: createCollection()command as follows:
    • This will create a collection called 'india' in which we can store info related to indian users as documents.
    • Ok message gets displayed if your collection was created successfully.
  6. Now to check whether the collection was created successfully, you can use the following command:
    • show collections lists all the collections within the database currently in use.
    • Create another collection on your own by following the same process.
