Creating Routes Using Express

Overview

Objective is to create many routes directing the client to different pages on the server.

What's needed:

  • Node and NPM installation
  • Installation of Express NPM package

How will our Routes look?

/services is basically one of the routes we will be creating.


How to Create Different Routes Using Express?

Routes represent the locations in which our website's or web app's pages get published.They begin with a '/' in the front. And are followed by the name for that specific route. Example: /contact. '/' alone represents your home page. Here you can publish the page that you want to be rendered by default.

Now, with a typical website in mind, let's create 4 different routes for displaying home, services, about and contact pages

  1. Create your express server and have it running at port 3000.

    2. To keep things simple, we are not going to deal with any post method here. Our 4 routes will only render pages to the client and not accept any input from them.

  2. Create 4 app.get() methods in your index.js file.
  3. Give different names to the routes. Ex: '/', /services, /about, /contact

To keep things simple, we won't be rendering actual html pages, but simple messages using res.send() methods.

Display Messages From Configured Routes Using res.send()

  1. Add res.send() method within each of the app.get() methods you have set up.
  2. For the message, you can display something like shown in the 1st image.Something which indicates to you that content is indeed coming from that route.
  3. Save the file and run your server.

Testing Out The New Routes

Once your server is up and running successfully

  • After the '/' in the URL (which should now be displaying whatever message you configured earlier for the home page), enter the route names you configured one after the other and check whether the messages you configured for them are being displayed correctly.

In our next lesson, we will see how to render an actual html page in a specific route.

When it comes to routing, express makes use of what is called 'routers' to do all the routing. app.get() methods are reserved for other things. This example was given just to make you understand the basics of routing.
