Creating a Simple Server Using Express

Overview

Objective is to create a simple server that renders 'hello world' on the browser when it is run.

What's needed:

  • Node and NPM installation
  • Installation of Express NPM package

How will our Simple Server look?

Our simple server will run on port 3000 and will render 'hello world' on the browser.


You should have installed node (as well as Npm module) in your system for the program to work.

How to Create the Server Using Express?

Create a separate folder called simpleServer and within it do the following:

  1. Using command line navigate into the simpleServer folder and run the following command:
    2. npm install express

    Doing so will install express in your working folder

  2. Now create a file called index.js. This is where you will code your server.
  3. Import the express module you just installed within the file and store it in a variable called 'express'.
  4. Transfer all the properties of express to another variable called 'app'.

    5. Variable app now basically represents our server.

How to Designate a Port Number for our Server?

Our server has to run (or listen in node terminology) on some port for it to be able to read requests from clients and send appropriate responses back to them.

app inherits the properites of express module. Therefore now it has access to '.listen' method of express. Using this we are passing in 3000 as the port number on which the server should be run. What follows the port number is a function that gets executed everytime a client tries to acccess our server on the given port. In our case, the function simply console.logs the words 'server running @ 3000' for now.

It is not necessary that the port number should be followed by a function. It can even be left empty. The code provided here is just an universal example which helps us understand that our server is up and running.

The function takes in 2 parameters: request and response. Servers receive requests and send responses. These are being passed as the parameters to the function.

The function itself is a fat arrow modern javascript function. You can also use the old javascript function syntax, though it is no longer recommended.

What do we want our Server to do?

We want our server to display the words 'hello world' on the browser. How can we accomplish this?

  • Server operations (or methods) in node can be broadly classified into 2 types: get and post. get is used whenever we want our server to get a page and display the same to the client. post is used whenever we want the client to submit (or post) something to the server.
  • Be it get or post, the syntax followed is pretty much same to app.listen method we just created. Only difference is that the app.listen gets replaced by app.get or app.post (depending on our requirements) and the port number is replaced by the route which we want the server to serve to the client.

In our case, since we just want to send across the words 'hello world' to the client's browser, we'll be making use of the app.get() method.

'/' always represents the home route. Which in our case is localhost:3000. And since we are not going to be receiving anything from the client here and only going to send a response, we will make use of the .send method that belongs to the express module and is responsible for sending messages to the client.

So our server is now ready to send the words 'hello world' to anyone who wishes to access it on localhost:3000

Now how do we run our server?

How to Run our Simple Server?

It's simple. All you have to do is navigate to the simpleServer folder using command line and execute the following code.

node index.js

And you'll see that our server is up and running now, displaying the words 'hello world' on the browser and the message 'server is running @ 3000' in the console window.
