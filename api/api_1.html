What are Web APIs?

In simple terms, APIs (which stands for Application Programming Interface) are data-loaded URLs (or endpoints) that allow one web service to access the data of another web service and present the same to the client. APIs present a win-win situation for web services to collaborate and grow their business together.

Travel websites like tripadvisor serve as good examples for illustrating API usage. Right from helping you book your favorite hotel to booking your flight ticket, they help you do everything needed for a good holiday on their website. How do they do this? By making use of APIs offered by those airlines and hotels that they liase with.

Universal logins (or single sign ons) are another common example that illustrate api usage. Apps that make use of universal logins - like google, facebook, twitter etc. - to allow users to subscribe to their services are basically handing over the user authentication process to these trusted 3rd party sources. How is this made possible? Through the use of APIs.

How Do APIs Work: An Overview

APIs operate on request-response basis, just like the web.

  1. Client contacts the server requesting some data.
  2. Server in turn contacts the 3rd party web service's API for that data.
  3. 3rd party web service authorises the server request based on what is known as the API key.
  4. Once the request is authorized, the API data is released by the 3rd party service as the response.
  5. This response is then relayed by the server to the client who made the request.

Why API Documentations Are Necessary?

APIs vary from web service to web service. They do not come in 'one-size-fits-all' packs. So if a web service has decided to build and release APIs (be it free or paid), then it becomes imperative that those APIs get documented properly. Only then the subscribers to those apis will be able to use them properly.

Who's the Target Audience?

The developers of the web service that subscribes to the apis.
