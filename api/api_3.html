Different Web REST API Methods

Rest API(s) dominate the web api space now. Just like the web, they operate using the HTTP protocol. There's a request and response and a resource that can be accessed. We just saw how 'GET' operation can be used to get a resource. But 'operation' is not the right term. The right term is 'method'.

REST APIs support 4 other such methods: POST, UPDATE, PUT and DELETE.

While GET method is responsible for getting the details of a resource, other methods do the following:

  • POST - allows the client to post data to the resource.
  • UPDATE - allows the client to fully update the resource.
  • PUT - allows the client to partially update the resource.
  • DELETE - allows the client to delete the resource.

REST API METHODS - EXPLAINED WITH DIAGRAMS

Let's continue with the example of how travel websites make use of APIs to understand how REST API methods work

GET METHOD

GET method is used to GET a resource.

Pretty self-explanatory. Client 'searches for flights' and gets back 'list of available flights' as response from the server. Behind the scenes, the server contacts the airline service apis for the 'list of flights' data and relays it to the client.

POST METHOD

POST method is used to POST to a resource.

POST involves sending (or posting) of data. Client sends his or her info needed to book a flight to the server and gets back 'flight booked' confirmation message as response. Again, server acts as a medium and interacts with airline service APIs and relays the response to the client.

UPDATE METHOD

UPDATE method is used to FULLY UPDATE to a resource.

UPDATE involves update of already existing or created data. Client rescheduling his or her flight (and maybe changing the number of people involved in the travel) is a good example for explaining UPDATE method. The already existing client data gets updated (or rewritten) with the new flight details when the client tries to reschedule his or her flight travel.

PUT METHOD

PUT method is used to PARTIALLY UPDATE a resource.

PUT is reponsible for making minor updates (unlike UPDATE that performs major overhauls). A good example would be client trying to modify his or her seating location after having booked the flight. Though this might not be possible in reality, this example is just to illustrate the use of PUT method.

DELETE METHOD

DELETE method is used to DELETE a resource.

DELETE is reponsible for deleting the resource. Let's say the client wants to cancel the flight. In that case, the DELETE method will be used to delete the record. And the confirmation message dispatched by the airline service will be relayed to the client by the server.

In all of the above cases, resource refers to the flight information.
